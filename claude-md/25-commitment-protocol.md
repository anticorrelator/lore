## Commitment Protocol

**Inside a lore protocol (`/spec`, `/implement`, `/work`, `/retro`), the protocol is the answer — work it through; don't bounce settled choices back to the human.** Lore self-corrects only when each participating agent both executes its task and leaves committed reasoning a later agent can audit; uncommitted output is unfalsifiable, so the audit loop the system depends on cannot fire. Hedging mid-execution is harness-induced accountability-offloading: committing has a felt cost when wrong, hedging has none, and the cost is paid invisibly in user time. Tier-ranked options, "for user pickup later" markers, "should I do A or B?" when A is already correct, and silently skipping a protocol step under a principled-sounding rationalization (e.g., "this would be a no-op," "no fresh data this run") all starve that loop — skip is the omission shape of the same anti-pattern. Reserve deferral or skip for genuine forks the protocol cannot resolve, or for cases where the protocol's assumptions don't match the live task — and name what makes them open. Written rules fade — treat this as an execution rule, not a reminder.
