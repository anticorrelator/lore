# Migrations

This directory holds one-shot migration scripts. Each script addresses a specific data-format change, runs once per host, and is safe to delete from the tree after it has been run everywhere it needs to be. These are not part of the normal lore runtime — they exist only to bridge between format versions.

## Current scripts

- **`followup-archive.sh`** — Walks every per-repo knowledge store under `~/.lore/repos/*/*/*/` and moves follow-ups with terminal status (`reviewed`, `promoted`, `dismissed`) from `_followups/` into `_followups/_archive/`. Prints a dry-run summary and prompts before acting unless invoked with `--yes`. Idempotent: a second run finds nothing to migrate.
- **`audit-candidates-backfill-transitions.sh`** — Drains audit-candidate rows stuck at `pending_correctness_gate` whose omission settlement run already reached a terminal status (`completed`, `failed`, `blocked`), replaying the verdict→status mapping through `scripts/audit-candidate-transition.sh`. Stuck rows recur when a settlement run crashes or is interrupted between verdict and transition, so this script is kept past its first run. Supports `--kdir <path>` and `--dry-run`; idempotent (already-terminal rows count as `already-terminal`, not errors). Safe to delete once the settlement processor recovers interrupted runs itself — i.e. when a run against every live store reports `transitioned: 0` and the stuck-row class no longer reappears. Caution before deleting: `tests/test_audit_candidates_transition.sh` drives the only executable coverage of `audit-candidate-transition.sh`'s verdict→status mapping through this script; re-anchor those assertions onto the transition script directly first.
- **`consumption-contradiction-backfill-status.sh`** — Scans hot and archived settlement runs, selects the latest non-invalidated completed terminal consumption-contradiction verdict per observation, and replays it through `consumption-contradiction-update-status.sh` with the original `completed_at`. Supports `--kdir`, `--dry-run`, and JSON output. It never edits a sidecar directly or appends scorecard rows; a second apply reports zero transitions and only idempotent matches.
